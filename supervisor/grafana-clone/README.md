# Reproducible local clone of the Raspberry Pi Grafana

Rebuilds a local Grafana with the **same dashboards and datasources** as a remote installation,
from version-controlled provisioning files. No `grafana.db` copy, no secrets in git.

## Layout

```
grafana-clone/
├── docker-compose.yml          # local Grafana, mounts ./provisioning
├── export-from-grafana.py      # pulls dashboards + datasources from the remote Grafana API
├── Makefile                    # make export / up / down
├── .env.example                # copy to .env, fill secrets (gitignored)
└── provisioning/
    ├── datasources/
    │   └── datasources.yaml     # GENERATED by export (UIDs pinned)
    └── dashboards/
        ├── dashboards.yaml      # file provider config
        └── json/<uid>.json      # GENERATED by export (one per dashboard)
```

## One-time setup

1. On the remote Grafana installation where we want to extract: Administration → Service accounts → add one with role
   **Admin** (required to list datasources) → create a token.
2. `cp .env.example .env` and fill `GRAFANA_URL`, `GRAFANA_TOKEN`, and the
   `INFLUXDB_*` values.
3. Pin the image in `docker-compose.yml` to the **same Grafana version** as the
   remote installation (check version running `curl -s http://<remote-grafana-IP>:3000/api/health` on the remote Grafana).

## Run

```bash
make export   # Grafana API -> provisioning files (commit these)
make up       # http://localhost:3000
```

Re-run `make export` whenever the Grafana dashboards change, review the diff, commit.

## What is and isn't cloned

- Cloned: datasource structure (UIDs pinned so dashboards resolve) and all
  dashboards.
- Not cloned: datasource **secrets** (the API never returns `secureJsonData`),
  alert rules, contact points, users, library panels, folders.

## Secrets

Secrets are injected at runtime via Grafana's `${VAR}` interpolation in the
provisioning files, sourced from `.env`. The exporter pre-templates InfluxDB
(`password` for v1). For any other datasource that needs
auth, add its secret keys to `datasources.yaml` as `${ENV_VARS}`.

## Data

This clones Grafana config only, not the time-series data. For panels to show
data, `INFLUXDB_URL` must reach an InfluxDB that has it:
- **Remote InfluxDB** — quickest, real data, read-only safe.
- **A local InfluxDB** — fully offline, but you must load/seed data yourself.

## Conventions

- Do export with `/api/dashboards/uid/:uid` (raw model). Do **not** use the UI
  "Export for sharing" JSON — it rewrites datasources into `__inputs` and breaks
  file provisioning.
- Do keep `allowUiUpdates: false` so dashboards stay file-driven. Edit in the
  UI only to then re-export.
